# 项目技术细节

## 架构

```
Client App → Mail Gateway (Next.js on Vercel) → External Mail Services
                ├── Admin Panel (/admin)
                ├── API Endpoints (/api/v1/*)
                └── Vercel KV (Redis)
```

## 安全架构

### 五因子 API 认证
1. **API Key**：标识客户端（KV 中加密存储）
2. **TOTP**：30 秒有效期的 6 位一次性密码（允许 ±1 时钟漂移）
3. **Config Key**：标识使用的邮箱配置（通过 x-config-key 头传递）
4. **请求签名**：HMAC-SHA256 签名（method + path + query + nonce + timestamp + body）
5. **Nonce**：最少 16 字符随机字符串，5 分钟内有效

### 加密体系
- **HKDF 派生**：使用 SHA-256，info 标签分 `aes-encryption-key` 和 `hmac-signing-key`
- **AES-256-GCM**：随机 12 字节 IV，附带认证标签，格式为 `iv.encrypted.authTag`

### 管理面板安全
- 三级安全等级（0: 仅密码, 1: +TOTP, 2: +DNS 应急）
- CSRF 保护：Double Submit Cookie
- 速率限制：登录 5 次/15分钟，应急 3 次/小时
- 会话 30 分钟 TTL

## 关键设计决策

### 配置缓存
- 使用内存缓存 + 版本号机制减少 KV 读取
- 版本号变更时自动失效缓存

### API Key 管理
- 最多 3 个活跃 API Key
- 第 4 个创建时自动废弃最旧的
- 废弃 Key 仍有 24 小时有效期
- 不能撤销最后一个活跃 Key

### 速率限制
- API 请求：10 次/分钟/Key
- 接收邮件：1 次/10 秒/ConfigKey
- 管理员登录：5 次/15 分钟/IP
- 应急登录：3 次/小时
