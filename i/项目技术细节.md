# 项目技术细节

## 架构
```
Client App → Mail Gateway (Next.js) → External Mail Services
                ├── Admin Panel (/admin)
                ├── API Endpoints (/api/v1/*)
                └── Vercel KV (Redis)
```

## 安全架构

### 五因子 API 认证
1. **API Key**：标识客户端（KV 中加密存储）
2. **TOTP**：30 秒有效期的 6 位一次性密码
3. **Config Key**：标识邮箱配置（x-config-key 头）
4. **请求签名**：HMAC-SHA256 签名
5. **Nonce**：最少 16 字符随机，5 分钟内有效

### 加密体系
- **HKDF 派生**：SHA-256，info 标签 `aes-encryption-key` / `hmac-signing-key`
- **AES-256-GCM**：随机 12 字节 IV，格式 `iv.encrypted.authTag`
